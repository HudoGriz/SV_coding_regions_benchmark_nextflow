Migrate from local bgzip_tabix modules to nf-core tabix/bgziptabix

## Summary
Replaced local bgzip_tabix and tabix_vcf modules with the standardized
nf-core tabix/bgziptabix module for better maintainability and features.

## Changes

### Main workflow (main.nf)
- Updated includes to use `TABIX_BGZIPTABIX` from nf-core
- Changed output channel references from `.out.vcf` to `.out.gz_index`
- Process invocations remain unchanged (using aliases)

### Configuration (conf/modules.config)
- Removed local module container specifications
- Added `ext.args2 = '-p vcf'` configuration for tabix
- Simplified to use wildcard pattern `BGZIP_TABIX.*`

### New files
- Added nf-core tabix/bgziptabix module files
- Updated modules.json with new module entry

## Benefits
- Community-maintained, tested module
- Multi-threading support for faster compression/indexing
- CSI index support for large chromosomes
- Automatic updates via nf-core tools
- Modern htslib version (1.21)

## Backward Compatibility
- Process names unchanged (using aliases)
- Output structure identical (meta, vcf.gz, tbi)
- Configuration patterns still work
- PublishDir patterns still match

## Testing
Configuration validated successfully with test_nfcore profile.

Related modules that can be removed:
- modules/local/bgzip_tabix.nf (no longer used)
- modules/local/tabix_vcf.nf (was never used)
